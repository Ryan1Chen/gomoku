# ♟ 五子棋 PWA

一個用純 HTML / CSS / JavaScript 寫成的五子棋遊戲，支援 PWA（漸進式網頁應用程式）。

---

## 🎮 功能特色

### 對弈功能

| 功能 | 說明 |
|------|------|
| 🤖 人機對戰 | 預設模式，內建 AI 對手，開啟即可直接開始 |
| 👥 雙人對戰 | 兩人輪流在同一台裝置下棋 |
| 🌱⚔️🔥 三段難度 | 新手 / 正常 / 高手，難度不同 AI 策略也不同 |
| 💡 建議下一步 | 分析當前局面，在棋盤上標示最佳落點並說明原因 |
| ↩ 悔棋 | 可反悔上一手（人機模式會同時撤回 AI 的棋） |
| 🏆 計分板 | 自動累計每局勝負，跨局保留 |
| 📲 離線遊玩 | 透過 Service Worker 快取，沒有網路也能玩 |

### AI 難度說明

| 難度 | 策略 | 適合對象 |
|------|------|---------|
| 🌱 新手 | 只防即將連五，其餘隨機落子 | 第一次玩五子棋 |
| ⚔️ 正常 | 攻守兼顧的全盤評估 | 有一點基礎的玩家 |
| 🔥 高手 | 威脅優先 + 一步前瞻，模擬對手反應後再決策 | 想挑戰自我的玩家 |

### 建議功能說明

按下「💡 建議」按鈕後：
- 棋盤上出現**藍色閃爍圓圈**，清楚標示建議落點
- 右側顯示**說明卡片**，解釋建議原因

| 等級 | 圖示 | 觸發條件 |
|------|------|---------|
| 立即獲勝 | 🏆 | 落子即連五 |
| 緊急防守 | 🛡️ | 對手下一步可連五或形成活四 |
| 強力進攻 | ⚔️ | 可形成活四、衝四或活三 |
| 最佳落點 | 💡 | 綜合攻守評估的最優位置 |

### 📖 教學系統

共 **12 堂課**，分 4 大類，每堂課有多個步驟，附迷你棋盤圖示說明：

| 類別 | 課程數 | 內容 |
|------|--------|------|
| 🎯 基礎棋步 | 2 堂 | 勝利條件（橫直斜）、先手後手概念 |
| ⚔️ 攻擊棋形 | 4 堂 | 活四、雙三、衝四、四三勝 |
| 🛡️ 防守策略 | 2 堂 | 防守時機與威脅判斷、封堵技巧 |
| 🏁 開局棋陣 | 4 堂 | 天元開局、花月、溪月、常見新手錯誤 |

學習進度會**自動儲存**在裝置上，關閉 App 重開後仍保留。

### 版面設計

- **桌機 / iPad 橫向**：左欄按鈕 ｜ 中央棋盤 ｜ 右欄資訊（輪次 + 建議卡片）
- **手機 / 直向**：自動切換為垂直排列，操作不受影響

## 📁 檔案結構

```
gomoku/
├── index.html      ← 遊戲主體（所有功能都在這一個檔案裡）
├── manifest.json   ← PWA 設定（App 名稱、圖示、顯示方式）
├── sw.js           ← Service Worker（讓遊戲可以離線執行）
└── README.md       ← 這份說明文件
```

## ❓ 常見問題

**Q：建議功能在 AI 下棋時可以用嗎？**

A：建議功能只在**輪到人類玩家**時可以按，AI 思考中或雙人對戰時都可以使用。

---

**Q：教學的學習進度會消失嗎？**

A：進度存在瀏覽器的 `localStorage`，只要不清除瀏覽器資料就會一直保留。
若從主畫面開啟（PWA 模式），資料同樣會保留。

## 🛠️ 技術說明

本專案使用純前端技術，**不需要伺服器、不需要資料庫**：

| 技術 | 用途 |
|------|------|
| HTML5 Canvas | 繪製棋盤、棋子、建議標記動畫 |
| Vanilla JavaScript | 遊戲邏輯、三段 AI、建議引擎、教學系統 |
| CSS3 | 三欄式響應式排版、動畫、漸層 |
| localStorage | 儲存教學學習進度 |
| Web App Manifest | PWA 設定（名稱、圖示、全螢幕） |
| Service Worker | 離線快取所有遊戲資源 |

**瀏覽器支援：** iOS Safari 11.3+、Android Chrome 40+、桌面版 Chrome / Firefox / Edge
